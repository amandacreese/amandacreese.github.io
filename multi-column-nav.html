Multi Column Navigation Documentation

About Multi Column Navigation

We have introduced usage of multi-column navigation in order to rectify issue of navigation going out of canvas when school had a lot of pages under one navigation topic.

Multi Column Navigation allows to overwrite styling of the navigation by adding a class to the drop down when there are more than (by default) 6 pages under a navigation topic. This styling will only apply to that particular topic, not all of them. Topics which have 6 or less pages will remain to have their original styling.

It is using a JavaScript function and CSS styling that can be changed if necessary. JavaScript function has built-in functions that allow you to change the targetting as well as classes, when the code should trigger etc. The core functionality of the code is available in buildAssets and barebone template snippet can be found on PrimarySite Team CodePen.

General Information

Barebones have been optimised to support column navigation drop down from 6+ pages up to 30 pages. Anything above is not accounted for by the barebones, however script includes an additional class if there are more than 30 pages within the navigation topic drop down which enables person implementing the code to do additional changes on the drop downs that have more than 30 pages.

CSS Explanation:
Script adds a custom class to the navigation topic drop down. It has been implemented using CSS grid and it overrides the existing styling of the navigation by resetting drop downs positioning and separating the pages into 3 individual columns.

Default CSS gives appropriate gaps, padding and sticks the navigation to the right hand side. All of the scripts styling can be changed to match the build.

Default CSS also includes a simple height based media query which supports up to 30 pages up to 600px viewport height.

JS Explanation:
Scripts entire core functionality is stored within buildAssets and it can be pulled using the script include. The only thing that needs to be put on the side are the options and calling of the function or alternatively just calling the function.

If you call in the navigation function multiColumnNav(); by default options are as follows:

  • targetItem: “.main-nav > ul >li >ul” Targets navigation topic drop down
  • pageCount: 6 Sets the amount of pages before going into columns
  • maxPageCount: 30 Sets the amount of max pages and adds additional class to the drop down
  • navParentClass: “nav-extended-reset” Sets class name for parent of the drop down (li)
  • navClass: “nav-extended” Sets class name for drop down UL
  • navClassMax: “nav-extended-max” Adds class name for drop down which meets maxPageCount criteria

Barebone CodePen snippet contains options by default, these can be changed if needed. All options are commended appropriately and explain their purpose.

Implementation

(Resources and examples can be found in the “Resources” section above.)

To implement multi column navigation on your build you will require the following:

  • Script include of the core functionality
  • Barebone CSS/JS only

1 - Go to Barebone CSS/JS only codepen and copy the JavaScript section.
2 - Open theme editor and create custom.js file if it doesn’t already exist.
3 - Paste in JS code copied earlier inside of custom.js file and save.
4 - Open base.html and paste in the script include nav-column-min.js in the scripts section.
Make sure that it sits above custom.js otherwise you will call a function that isn’t available yet.
5 - Go back to the Barebone CSS/JS only codepen and copy all of the CSS styling.
6 - Open style.css and underneath all of the navigation styling paste in CSS code from the CodePen.
7. All done You might need to tweak the styling depending on your build
