Swiper Implementation.

About 'Swiper' and 'Flick'.

We introduced usage of swiper in order to modernise our responsive and desktop designs. Main task for swiper is to enable 'swipe' functionality on touch screens as well as provide user with button, thumbnail and dot navigation. We have extended functionality by introducing Flick .

Think of Flick as bridge between swiper and build team. Flick allows us to initiate multiple swiper instances with separate settings across the site. Building navigation buttons and injecting classes on dynamic elements ( updates areas ) is an added extra. Flick does not limit, change or stop any native functionality that comes with swiper.

This documentation covers information regarding Flick .

Any information regarding native swiper functionality and options can be found in swiper official documentation .

PS settings:

Table of currently available options within swiper instance. All options should be placed in ps_settings object inside swiper instance object:

  • Paramater
  • Type
  • Default
  • Example
  • assignEle
  • Array
  • NA
  • assignEle : ['#blockBravo>ul', '#blockBravo>ul>li']
  • Array should contain two items. First item is selector for element that becomes swiper-wrapper, second item is selector for elements that become swipe-content
  • initWhen
  • string/Number
  • global
  • initWhen: 'global' or initWhen : number
  • Defines when swiper instance is initalized. number should be replaced with pixel value of when swiper initializes. 'px' should be ommited

Following options apply to nav object which should be set within ps_settings if you need PS arrows

  • Paramater
  • Type
  • Default
  • Example
  • appendTo
  • Array
  • NA
  • appendTo : ['.prev-arrow', '.next-arrow'] or ['.arrow-block']
  • Array contains selectors where navigation buttons are injected. 2 array items represent buttons on either side of the swiper instance. First - for the "previous" and second for the "next". One array item means that arrows are to be injected inline - side by side in element reflected in the array
  • btnClass
  • string
  • swiper-btn
  • btnClass : 'swiper-btn'
  • Extra class name to be added to both arrows for styling purposes. Multiple classes should be divided by a space . Left button has a class 'nav-prev' and right button has a class of nav-next regardless of 'btnClass' option
  • prevHtml
  • string
  • Prev
  • prevHtml : 'Previous'
  • String that appears within left ( navigating back ) navigation button. can be any HTML. Pass icon reference to use font icon
  • nextHtml
  • string
  • Next
  • nextHtml : 'Next'
  • String that appears within right (navigating forward ) navigation button. can be any HTML. Pass icon reference to use font icon

Error Messages:

We have implemented some error handling within swiper that are connected to swiper initialization. Errors are shown within Swiper instance block as well as in browser console.

Please find registered errors and solutions below. Native swiper or general syntax errors will not be shown within swiper instance. Those will appear in console only.

Assuming swiper instance with id swiperOne .

#swiperOne: Swiper instance : 'swiperOne' exists in the DOM, but key is missing from 'swiperParams' object

Script found swiper-container with id in the DOM, that was not initialized within swiperParams object in flick-variable.js

#swiperOne: : 'initWhen' must be 'global' or number

initWhen has wrong value applied to it. It has to be 'global' or 980 (pixel value without px and quotes). Example below shows an error where user initiated property with '1200px' - quotes and 'px' are redundant.

#swiperOne: : 'nav' object is set but 'appendTo' is missing.

nav property tells the script that user wants to build arrows. appendTo key value pair is needed to know where to inject buttons. Please refer to PS settings - appendTo option. and choose desired option

#swiperOne: : Missing information in 'assignEle'. Provide swiper wrapper and swipe content.

assignEle is used when user wants to set class names to elements we cannot access such as updates areas. This key value pair should have an array as value holding two items. First for the element that becomes swiper-wrapper and second that becomes swipe-content.

Example below shows a situation where user assigned swiper wrapper, but forgot to assign swipe content. Array has one item in it.

#swiperOne: : 'appendTo' has to be an array

This means that nav object has been declared and key appendTo has been set with wrong type of value. This has to be an array with one or two items.

#swiperOne: : selector '.selector-does-not-exist' cannot be found within the DOM

nav object is set. appendTo key is declared as an array. single selector provided within array does not exist. Please check if array item contains selector that appears within the DOM.

#swiperOne: : selector '.i-exist' or '.i-do-not-exist' cannot be found within the DOM

nav object is set. appendTo key is declared as an array. Separate navigation buttons selectors provided within array. One of these elements do not exists within the DOM. Please check if array item contains selectors that appears within the DOM.

#swiperOne: : Maximum amount of items in 'appendTo' is exceeded. Max is 2

appendTo array can consist of maximum 2 items ( one for previous button and one for next button . Error states that array has more than 2 items. Please check array.
